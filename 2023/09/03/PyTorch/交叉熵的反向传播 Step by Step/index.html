Step 1：A Simple Case

以交叉熵为例，普通的二分类问题

Binary Cross Entropy=−(ylog⁡(y^)+(1−y)log⁡(1−y^))\text {Binary Cross Entropy} = -\left (y \log (\hat {y}) + (1-y) \log (1-\hat {y})\right) Binary Cross Entropy=−(ylog(y^​)+(1−y)log(1−y^​))

Multiclass Cross Entropy=−∑c=0Cyclog⁡(y^c)\text {Multiclass Cross Entropy} = -\sum_{c=0}^{C} y_c \log (\hat {y}_c) Multiclass Cross Entropy=−c=0∑C​yc​log(y^​c​)

其中，yyy 表示真实的标签值（通常是one-hot向量），y^\hat{y}y^​ 表示模型的输出，CCC 表示有多少类别需要分类

  1. 由于模型输出并不是单纯的概率值，因此会对模型输出进行softmax计算，得到y^\hat{y}y^​

    softmax(xi)=exi∑j=1nexj\text {softmax}(x_i) = \frac {e^{x_i}}{\sum_{j=1}^n e^{x_j}} softmax(xi​)=∑j=1n​exj​exi​​

  2. 再以简单的单分类任务来说，
    1. yyy = [1, 0, 0], y^\hat{y}y^​ = [0.5, 0.2, 0.3] -> log(y^\hat{y}y^​)=[-0.9398, -1.2398, -1.1398]
    2. 计算时，只有c=0时，结果有值（-0.9398），其他类别时为0，无需计算

梯度

  1. 结合前面的计算，每次计算loss时，实际上只有对应类别的计算是有效的，因此将其简化为−yclog⁡(y^c)-y_c\log(\hat{y}_c)−yc​log(y^​c​)，而由于yc=1y_c=1yc​=1，所以其实只有−log⁡(y^c)-\log(\hat{y}_c)−log(y^​c​)

  2. 这个公式只需要计算log的导数。对于f(x)=log⁡(x)f(x)=\log(x)f(x)=log(x)，其导数f′(x)=1xf^{'}(x)=\frac{1}{x}f′(x)=x1​。

    1. 当 i=ji = ji=j 时：

    ∂yi∂xi=exi(∑k=1nexk)−exiexi(∑k=1nexk)2=yi(1−yi)\frac {\partial y_i}{\partial x_i} = \frac {e^{x_i}\left (\sum_{k=1}^n e^{x_k}\right) - e^{x_i} e^{x_i}}{\left (\sum_{k=1}^n e^{x_k}\right)^2} = y_i (1-y_i) ∂xi​∂yi​​=(∑k=1n​exk​)2exi​(∑k=1n​exk​)−exi​exi​​=yi​(1−yi​)

    ∂yi∂xj=−exjexi(∑k=1nexk)2=−yiyj\frac {\partial y_i}{\partial x_j} = -\frac {e^{x_j} e^{x_i}}{\left (\sum_{k=1}^n e^{x_k}\right)^2} = -y_i y_j ∂xj​∂yi​​=−(∑k=1n​exk​)2exj​exi​​=−yi​yj​

autograd

在PyTorch内部有一套完善的求导机制。因此，仅需按照其规则进行书写即可实现前向传播、反向传播（链式求导）。如下所示，在forward填写loss的计算，backward中填写grad的计算即可

Backward

这里的grad_output 表示梯度计算的输出。它是一个与前向传播函数的输出形状相同的张量。在反向传播过程中，我们将计算当前函数的导数，乘以 grad_output 作为输入。这个值代表了后续节点对当前节点的梯度贡献。

通过将 grad_output 乘以导数（也称为雅可比向量积），可以有效地传递梯度信息到较早的节点，从而实现自动微分。

注意，grad_output 的形状必须与函数的输出形状一致，否则会引发错误。

为什么

本文不介绍分布式训练的其他原理，仅关注模型最后的输出，以及损失函数的计算。

前文已经详细介绍了交叉熵的实现，那么为什么到分布式训练的时候，就不能直接复用呢？主要原因在于模型的输出被拆散了。回顾模型的输出，其shape为[bs, C]，bs表示每次训练的数据量batch size，C表示任务的总的分类数量，在LLM中主要是词表大小vocab_size（下文用此代表类别数量）。

对于模型的输出而言，分布式（假设有word_size张GPU）主要分两种情况

  1. 拆分bs，每张卡上有bs数据量，那么整合模型的最后输出应为[bs * word_size, vocab_size]
  2. 拆分vocab_size，单个模型的输出为[bs, vocab_size // word_size]。这种情况需要保证vocab_size能够整除word_size

因此，在进行loss计算前，需要先同步所有的模型输出，然后再进行loss计算。这里的同步，就是从所有GPU中获取模型的输出。但考虑到通信开销问题，有些操作是可以进行优化的，即无需通信全部内容再计算。重新思考损失的计算过程可以分为如下几步：

  1. SoftMax计算：exp⁡(x)/∑(exp⁡(x)\exp(x) / \sum(\exp(x)exp(x)/∑(exp(x)
  2. log的计算： −log⁡[exp⁡(x)/∑(exp⁡(x)]-\log[\exp(x) / \sum(\exp(x)]−log[exp(x)/∑(exp(x)]
  3. 索引：根据targets索引矩阵对应位置 x[...,...]x[..., ...]x[...,...]

其中，前两者跟最后一个步骤是可以独立运行的。而前两者的计算是耦合的，为节约通信量，对log计算进行变化

  1. −log⁡[exp⁡(x)/∑(exp⁡(x)]-\log[\exp(x) / \sum(\exp(x)]−log[exp(x)/∑(exp(x)]
  2. −log⁡[exp⁡(x)]+log⁡[∑exp⁡(x)]-\log[\exp(x)] + \log[\sum\exp(x)]−log[exp(x)]+log[∑exp(x)]
  3. log⁡[∑exp⁡(x)]−x\log[\sum\exp(x)] - xlog[∑exp(x)]−x

因此，可以由原始的func1推导至分布式的func2
